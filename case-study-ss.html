Sneaker Shop

Building e-commerce the React and Redux way

The Challenge

Over the past two years, React.js has had a growing interest in the development community. Even more so than Angular.js. Buzzwords and phrases like "It's the V in MVC", "Component driven", "Unidirectional data flow" are used to describe React.js. After building simple apps in React.js, I wasn't entirely sure how it would scale as the app got complicated especially with regards to state management. The most popular solution to managing state is called Redux. I wanted to see what it was all about.

The Solution

Redux is known to be Flux-y. Flux is a data flow architecture proposed by Facebook that could be ideal for React. Redux uses Flux concepts like stores, reducers and actions to manage state in React Components. I wanted to try this in a simple e-commerce website to see it in action.
